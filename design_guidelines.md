# 恋爱互动小程序设计指南

## 1. 品牌定位

**应用名称**：心动助手

**应用定位**：线上认识线下见面关系推进助手，帮助用户管理多个潜在发展对象，智能推荐互动内容，加速推进到恋人关系

**设计风格**：清新、专业、高效、温暖

**目标用户**：
- 相亲人群
- 交友App用户转线下
- 社交场合认识心仪对象的人

**核心场景**：
1. **相亲** - 信息完整、目的明确、时间有限
2. **搭讪** - 信息很少、需要快速建立联系
3. **交友App线下见面** - 有线上聊天基础
4. **聚会社交** - 有共同朋友、氛围轻松
5. **职场/学校** - 长期接触机会
6. **兴趣活动** - 有共同爱好

## 5. 组件使用原则

### 组件选型约束
- 按钮 → `Button`
- 输入框 → `Input` / `Textarea`
- 卡片 → `Card`
- 标签 → `Badge`
- 弹窗 → `Dialog`
- 开关 → `Switch`
- 进度条 → `Progress`
- 单选 → `RadioGroup`

### 页面结构
```
页面容器 (min-h-screen bg-gray-50)
  └─ 内容区域 (p-4)
      └─ Card 组件 (rounded-2xl shadow-sm)
```

## 7. 核心功能设计

### 建立档案流程
1. 基本信息：姓名、年龄、职业
2. 见面场景：选择相遇场景
3. 初印象：评分和印象标签
4. 兴趣爱好：选择兴趣标签
5. 智能推荐：根据信息生成推荐

### 智能推荐策略
- 相亲场景 → 快速破冰话题、深入了解问题
- 搭讪场景 → 开场白建议、轻松话题
- App见面 → 延续线上话题、约会建议
- 聚会场景 → 共同朋友话题、群体活动
- 职场/学校 → 渐进式任务、日常话题
- 兴趣活动 → 兴趣相关话题、活动邀约
