# 恋爱互动小程序设计指南

## 1. 品牌定位

**应用名称**：心动约会

**应用定位**：相亲/交友/线下见面互动助手，帮助情侣快速破冰、增进了解、推进恋爱进程

**设计风格**：温馨、浪漫、亲和、轻松、游戏化

**目标用户**：
- 相亲初次见面的男女
- 线上交友转线下的情侣
- 希望增进感情的情侣

## 5. 组件使用原则

### 组件选型约束

**通用 UI 组件优先使用 `@/components/ui/*`，禁止用 View/Text 手搓以下组件**：
- **按钮** → `Button` (主按钮粉色，次按钮灰色边框)
- **输入框** → `Input` / `Textarea` (表单场景)
- **卡片** → `Card` (内容容器)
- **标签/徽章** → `Badge` (状态标签、任务难度)
- **弹窗** → `Dialog` (确认操作、表单提交)
- **底部抽屉** → `Drawer` (筛选、详情展开)
- **标签切换** → `Tabs` (任务分类、进度切换)
- **进度条** → `Progress` (任务进度、关系进度)
- **开关** → `Switch` (设置项)
- **空状态** → 直接用 Card + 居中文字
- **加载态** → `Skeleton` (列表加载)

### 页面结构模板

```
页面容器 (min-h-screen bg-gray-50)
  └─ 顶部区域 (p-4)
      └─ 标题 + 副标题
  └─ 内容区域 (p-4 gap-4)
      └─ Card 组件 (圆角-2xl 阴影-sm)
          └─ 卡片内容
  └─ 底部操作栏 (fixed bottom-0 p-4 bg-white)
      └─ Button 组件
```

### 圆角规范

- 大卡片：`rounded-2xl` (16px)
- 小卡片/按钮：`rounded-xl` (12px)
- 标签/徽章：`rounded-full`

### 阴影规范

- 卡片悬浮：`shadow-sm`
- 按钮悬浮：hover 时 `shadow-md`

## 7. 核心页面设计要点

### 首页

- **顶部**：情侣头像 + 相识天数（大数字突出）
- **快捷入口**：4个入口卡片（开始互动、默契测试、打卡任务、美好时刻）
- **AI破冰话题**：每日推荐话题卡片，点击刷新

### 互动任务页

- **任务分类**：约会准备、聊天话题、互动游戏、浪漫举动
- **任务卡片**：标题、描述、难度标签、完成按钮
- **进度统计**：已完成/总任务数

### 默契测试页

- **题目类型**：选择题、判断题
- **答题流程**：一人出题 → 另一人作答 → 显示结果
- **结果展示**：默契度百分比 + 趣味评语

### 关系进度页

- **里程碑时间线**：相识、第一次约会、牵手、表白等
- **照片墙**：上传照片记录美好瞬间
- **进度条**：当前关系阶段可视化

### 我的页面

- **个人信息**：头像、昵称、性别
- **配对信息**：对方昵称、配对时间
- **设置项**：通知设置、隐私设置

## 8. 小程序约束

### 性能优化

- 图片使用懒加载 `lazyLoad`
- 长列表使用虚拟列表
- 避免一次加载过多数据

### 包体积

- 主包 ≤ 2MB
- 图片资源使用 CDN 或对象存储
- 非必要资源延迟加载

### 跨端兼容

- 所有垂直排列的 `Text` 添加 `block` 类
- `Input` 必须用 `View` 包裹
- 平台检测使用 `Taro.getEnv()`
